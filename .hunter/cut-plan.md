# Cut Plan: RefusalRail

Under deadline pressure, cut breadth first. Never cut credibility.

## Must keep

- Public live URL at `https://refusalrail.veithly.workers.dev`.
- Complete hero path: guest/test wallet, shock, unsafe agent attempt, saved NO receipt, receipt detail, and refresh persistence.
- Sponsor/domain primitive: Arbitrum Sepolia RefusalHub contract path, deployment addresses, and explorer proof.
- Runtime proof: Playwright cross-browser flow, mobile run, refresh persistence, second role/history proof, claim matrix, and realness audit.
- Honest states: empty receipt rail, loading/saving state, invalid tx hash error, not-found receipt recovery, safe OK contrast.

## Can cut now

- Extra marketing sections below the proof bento.
- Additional chart cards, animations, and non-essential dashboard filters outside the core loop.
- Profile, theme switcher, and longer architecture copy that do not change the refusal proof.

## Cut last

- Auditor receipt history, because it proves return value beyond the first run.
- Receipt detail proof table, because it turns the NO moment into a durable artifact.
- Arbitrum deployment/explorer proof, because sponsor technology is the visible trust mechanism.

## Why cut

The project protects the complete end-to-end workflow and removes breadth around it. The judge must still open the live URL, run the refusal, inspect a saved receipt, see the safe contrast, and verify the Arbitrum proof path.

## Risk if cut

Cutting public access, the core loop, sponsor proof, or runtime evidence would make RefusalRail look like a narrated demo shell instead of a publicly operable RWA safety product.
